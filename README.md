<details>
<summary><b>🐍 Snake not showing? — Complete Fix Guide (click to expand)</b></summary>

<br/>

**Why it's not working yet:**
The snake needs a GitHub Action to run once, which creates an `output` branch with the SVG files. Until that branch exists, the image URL shows a broken link.

**Fix in 4 steps:**

**Step 1 — Check your workflow file exists**
Go to `hassan0703/hassan0703` repo → look for `.github/workflows/` folder → confirm `snake.yml` is inside.

**Step 2 — Enable write permissions (most common fix)**
Repo → **Settings** → **Actions** → **General** → scroll to "Workflow permissions" → select **"Read and write permissions"** → click **Save**.

**Step 3 — Use this exact working workflow** (replace your current file if it failed):

```yaml
name: Generate Snake

on:
  schedule:
    - cron: "0 0 * * *"
  workflow_dispatch:
  push:
    branches:
      - main

jobs:
  generate:
    runs-on: ubuntu-latest
    timeout-minutes: 10

    steps:
      - name: Checkout
        uses: actions/checkout@v3

      - name: Generate snake
        uses: Platane/snk/svg-only@v3
        with:
          github_user_name: Hassan0703
          outputs: |
            dist/github-contribution-grid-snake.svg
            dist/github-contribution-grid-snake-dark.svg?palette=github-dark
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Deploy to output branch
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist
          publish_branch: output
          commit_message: "Update snake [skip ci]"
```

**Step 4 — Trigger manually**
Repo → **Actions** tab → **Generate Snake** (left sidebar) → **Run workflow** button → **Run workflow**.

Wait ~60 seconds. A green checkmark ✅ means it worked. The snake will now appear and auto-update daily.

**If you still get a red ✗ error:**
Go to Actions → click the failed run → click the failed step → read the error message. The most common cause is missing write permissions (Step 2 above).

</details>
